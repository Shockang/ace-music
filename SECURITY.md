# Security Policy

## Supported Versions

Security fixes are applied on a best-effort basis to the latest state of the
`main` branch. Older snapshots may not receive backports.

## Reporting a Vulnerability

Do not open a public issue for an undisclosed security vulnerability.

Send a private report to the maintainer through GitHub security advisories if
enabled on the repository. If private advisories are unavailable, contact the
maintainer directly through the repository owner profile and include:

- affected version or commit
- reproduction steps
- impact summary
- suggested mitigation if known

## What To Expect

Reports are reviewed on a best-effort basis. You may be asked for additional
details or a reduced reproduction case before a fix is prepared.
