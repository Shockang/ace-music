# ace-music 项目严格验收文档

> **工程唯一核心目标：** 从文本描述到音频文件的全自动化高质量音乐生成
> **外部依赖：** `auto-director`（消费 DirectorBridge 接口）、`MiniMax API`（可选云后端）、`StableAudio`（可选云后端）
> **验收方法：** 每次迭代后按本文档逐项验证，更新状态和日期，保留历史记录
> **当前版本：** v0.3.0 | 269 tests PASS | 全部通过

---

## 一、验收流程

每次迭代完成后，执行以下步骤：

1. **运行全量测试** — `pytest --tb=no -q`，记录通过/失败数
2. **运行代码检查** — `ruff check src/ tests/`，确认零 lint 错误
3. **CLI 冒烟测试** — `ace-music generate --mock` + `ace-music validate`，验证端到端可通
4. **逐项验证本文档所有矩阵** — 更新状态列
5. **提交文档** — 确保验收文档与代码同步

**门控规则：**
- 测试全部通过（0 fail）才能标记验收通过
- lint 零错误才能标记验收通过
- CLI generate + validate 冒烟测试通过才能标记验收通过

## 二、全链路验收矩阵

核心流水线拆解为 6 个阶段，每个阶段需验证代码存在性、Schema 合规和测试覆盖。

```
PipelineInput
  -> LyricsPlanner -> StylePlanner -> [Generator|MiniMax|StableAudio] -> PostProcessor -> OutputWorker
-> PipelineOutput
```

Note: EmotionMapper 在 StylePlanner 阶段内部调用，不是独立流水线阶段。

### 2.1 流水线阶段验收

| # | 阶段 | 模块 | 输入 Schema | 输出 Schema | 验证方法 | 对应测试 |
|---|------|------|------------|------------|----------|----------|
| 1 | LyricsPlanner | `tools/lyrics_planner.py` | LyricsInput | LyricsOutput | 结构化歌词解析 | test_lyrics_planner.py |
| 2 | StylePlanner | `tools/style_planner.py` | StyleInput | StyleOutput | 风格映射到 ACE-Step 参数 | test_style_planner.py |
| 3 | EmotionMapper | `tools/emotion_mapper.py` | AudioSceneContract | MappedAudioParameters | 场景情绪转音频参数（内部调用） | test_style_planner.py, test_pipeline.py |
| 4 | Generator | `tools/generator.py` | GenerationInput | AudioOutput | ACE-Step 本地生成 | test_generator.py |
| 5 | PostProcessor | `tools/post_processor.py` | PostProcessInput | ProcessedAudio | DSP 链（响度/格式/采样率） | test_post_processor.py |
| 6 | OutputWorker | `tools/output.py` | OutputInput | PipelineOutput | 文件写入 + 元数据 | test_output.py |

**判定标准：**
- PASS = 代码存在 + Schema 合规 + 测试通过
- PARTIAL = 代码存在但缺少测试或存在已知问题
- FAIL = 无代码或代码不完整

## 四、Provider 路由验收

FeatureRouter 支持 3 种生成路径，通过 `PipelineInput.backend` 字段路由。

| 路径 | backend 值 | 流水线阶段 | 依赖 | 验证测试 |
|------|-----------|-----------|------|----------|
| Local ACE-Step | `"acestep"` (默认) | LyricsPlanner -> StylePlanner -> Generator -> PostProcessor -> OutputWorker | torch, transformers, soundfile (optional) | test_generator.py, test_pipeline.py |
| MiniMax 云端 | `"minimax"` | MiniMaxMusicGenerator -> OutputWorker | MINIMAX_API_KEY | test_minimax_generator.py |
| StableAudio | `"stable_audio"` | StableAudioGenerator -> OutputWorker | torch, stable-audio-tools (optional) | test_stable_audio_generator.py |

**路由判定逻辑：**
- `backend == "minimax"` -> `_run_minimax_pipeline()`，跳过 style/lyrics/post_processor
- `backend == "stable_audio"` -> `_run_stable_audio_pipeline()`，跳过 style/lyrics/post_processor
- 其他 -> `_run_local_pipeline()`，完整 6 阶段流水线

**路由测试覆盖：** test_providers.py

## 五、Schema 合规验收

所有 Schema 类均基于 Pydantic BaseModel，定义在 `src/ace_music/schemas/` 下。

| # | Schema 类 | 模块文件 | 关键字段 | 验证规则 |
|---|----------|---------|---------|---------|
| 1 | AudioSceneContract | audio_contract.py | scene_id, duration_seconds, mood, mix | mood 非空，duration > 0 |
| 2 | AudioOutput | audio.py | file_path, duration_seconds, sample_rate, format | file_path 必须存在 |
| 3 | LyricsInput | lyrics.py | raw_text, language, is_instrumental | raw_text 非空 |
| 4 | LyricsOutput | lyrics.py | formatted_lyrics, is_instrumental, segments | segments 为列表 |
| 5 | MaterialContext | material.py | entries, is_empty | entries 列表 |
| 6 | OutputConfig | output_config.py | format, sample_rate, target_lufs | format 为 wav/mp3/flac |
| 7 | PipelineInput | pipeline.py | description, duration_seconds, backend | description 非空，duration 5-240 |
| 8 | PipelineOutput | pipeline.py | audio_path, duration_seconds, format, sample_rate, metadata | audio_path 非空 |
| 9 | StyleInput | style.py | description, reference_tags, tempo_preference | description 非空 |
| 10 | StyleOutput | style.py | prompt, guidance_scale, infer_step | prompt 非空 |

**合规测试：** test_audio_contract.py, test_preset_schemas.py, test_output_config.py
